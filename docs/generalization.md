## Generalization

DEALta's architecture — detect a delta, route it to domain perspectives, check it against policy, find cross-domain risks, produce a human-ready decision pack — is not specific to contracts. The same pipeline applies to code review, where a PR diff routes to security, performance, and architecture reviewers, checks against style and dependency policies, and flags compound risks like a database migration coupled with an API change. It applies to PRD review, where a spec change routes to engineering, design, and legal stakeholders, checks for consistency with existing commitments, and surfaces conflicts between new requirements and in-flight work. Vendor evaluation, compliance monitoring, incident response triage — any domain where a structured change needs multi-perspective review against accumulated context fits the pattern. The agents change names, the policy rules change content, but the orchestration skeleton — versioned state, delta detection, multi-lens evaluation, dependency tracking, escalation logic — transfers intact.

Running DEALta on a software license agreement would immediately surface three gaps. The policy rules are calibrated to travel supplier commercial terms: commission structures, SLA P1 response time definitions, booking volume triggers. A software license cares about IP assignment, indemnification caps, and source code escrow — none of which exist in the current policy config. The compound risk detection ground truth assumes a specific clause dependency pattern (rate changes interacting with volume commitments interacting with SLA penalties) that doesn't map to software licensing risk clusters. And the materiality thresholds reflect Nexus's commercial risk tolerance, not a universal standard. The architecture generalizes — agents, state schema, eval methodology, invalidation logic. The domain knowledge embedded in policy configuration and ground truth does not. Porting DEALta to a new contract type is a configuration and eval problem, not an architecture problem. That distinction is the point.
